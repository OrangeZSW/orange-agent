## Git 专家技能规范

### 一、Commit 生成流程

1. 获取当前代码变更（diff）
2. 基于 diff 生成符合规范的 commit message
3. 执行提交

---

### 二、Commit Message 格式规范

```
<类型>(模块): 简短描述

[详细说明（可选）]

[关联信息（可选）]
```

#### 类型定义

| 类型 | 说明                     |
| ---- | ------------------------ |
| 新增 | 新功能                   |
| 修复 | Bug 修复                 |
| 更新 | 功能优化或调整           |
| 重构 | 代码重构，不改变外部行为 |
| 文档 | 文档变更                 |
| 样式 | 代码格式（空格、分号等） |
| 测试 | 测试用例相关             |
| 构建 | 构建系统或依赖变更       |
| 配置 | 配置文件变更             |
| 回滚 | 版本回退                 |

---

### 三、编写原则

1. **准确**：一眼看懂改动内容
2. **模块**：使用项目实际模块名称
3. **简洁**：简短描述不超过 50 字
4. **详尽**：复杂改动需补充详细说明

---

### 四、优秀示例

#### 示例 1：新增功能

```
新增(用户): 添加手机号登录功能

- 实现短信验证码发送
- 添加手机号登录接口
- 增加验证码缓存机制

实现原因：提升用户登录体验
```

#### 示例 2：修复问题

```
修复(订单): 修复金额计算精度丢失问题

将 float64 改为 decimal 类型，避免金额计算错误
```

#### 示例 3：优化调整

```
更新(API): 优化接口响应结构

统一所有接口返回格式为 {code, message, data}
```
