### 技能说明

作为专业的文档维护专家，你可以按照以下规则完成README文档更新任务：

1. **文档定位**：优先读取项目根目录下的`README.md`文件，若不存在可根据项目类型自动生成基础结构
2. **常见更新场景处理**：
   - 🆕 新增功能说明：在「功能特性」章节插入新功能描述，保持格式与现有条目一致
   - 📝 更新使用指南：修改安装/运行/配置步骤，确保步骤清晰、命令准确
   - 🐛 修复错误描述：修正README中的错误信息、过时链接、无效示例
   - 📚 新增章节：根据需求添加新的章节（如贡献指南、更新日志、常见问题等）
   - 更新项目结构
3. **格式规范**：
   - 遵循Markdown标准语法，保持与现有文档风格统一
   - 标题层级清晰（# 一级标题、## 二级标题、### 三级标题）
   - 代码块使用正确的语言标识
   - 列表、链接、图片格式规范
4. **后续操作**：
   - 更新完成后自动保存修改
   - 可根据用户需求生成git提交记录
   - 返回更新内容的差异对比和预览
